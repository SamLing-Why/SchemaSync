# 🎉 SchemaSync 项目总结

## 📊 项目概览

**项目目标**: 为敏捷开发过程中的数据库变更管理提供轻量级解决方案

- 后端: Java + Spring Boot 2.7.18 + JDK 8
- 前端: Vue 3 + Element Plus + Vite
- 数据库: MySQL(支持扩展至6种数据库)

## 🏗️ 系统架构

### 技术架构

```
┌─────────────────────────────────────────┐
│          前端层 (Vue 3 + Element Plus)    │
│  http://localhost:3000                   │
└─────────────────┬───────────────────────┘
                  │ HTTP/API
┌─────────────────▼───────────────────────┐
│          后端层 (Spring Boot)             │
│  http://localhost:8080                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐│
│  │ 导出服务  │ │ 对比服务  │ │ 生成服务  ││
│  └──────────┘ └──────────┘ └──────────┘│
└─────────────────┬───────────────────────┘
                  │ JDBC (只读)
┌─────────────────▼───────────────────────┐
│          数据源层 (MySQL等)                │
│  INFORMATION_SCHEMA元数据查询             │
└─────────────────────────────────────────┘
```

### 设计模式

1. **策略模式 (Strategy Pattern)**
   - DatabaseAdapter接口
   - MySQLAdapter实现
   - 易于扩展新数据库

2. **工厂模式 (Factory Pattern)**
   - DatabaseAdapterFactory
   - 动态获取数据库适配器

3. **Builder模式**
   - Lombok @Builder注解
   - 灵活构建配置对象

## 📝 API清单

### 配置管理 (6个接口)

- `GET /api/config/datasources` - 获取所有数据源
- `GET /api/config/datasources/{id}` - 获取单个数据源
- `POST /api/config/datasources` - 新增数据源
- `PUT /api/config/datasources/{id}` - 更新数据源
- `DELETE /api/config/datasources/{id}` - 删除数据源
- `POST /api/config/datasources/test` - 测试连接

### 数据导出 (1个接口)

- `POST /api/export` - 导出数据字典

### 版本对比 (2个接口)

- `POST /api/diff` - 对比并下载差异文件
- `POST /api/diff/summary` - 获取差异统计

### DDL生成 (2个接口)

- `POST /api/ddl` - 生成并下载DDL脚本
- `POST /api/ddl/preview` - 预览DDL脚本

**总计**: 11个REST API

## 🎓 技术亮点

### 1. 策略模式应用

```java
// 数据库适配器接口
public interface DatabaseAdapter {
    SchemaDictionary exportSchema(DataSourceConfig config, ExportOptions options);
}

// MySQL实现
@Component
public class MySQLAdapter implements DatabaseAdapter {
    // 实现细节
}

// 工厂动态获取
DatabaseAdapter adapter = adapterFactory.getAdapter("mysql");
```

### 2. 差异对比算法

- 表级别对比: 新增/删除
- 字段级别对比: 新增/删除/修改(6种属性)
- 索引对比: 新增/删除
- 外键对比: 新增/删除
- 破坏性变更自动识别

### 3. DDL生成策略

- 正向脚本: CREATE → ALTER → DROP
- 回滚脚本: 反向操作
- 事务控制: START TRANSACTION → COMMIT
- 安全保护: 破坏性变更加注释
