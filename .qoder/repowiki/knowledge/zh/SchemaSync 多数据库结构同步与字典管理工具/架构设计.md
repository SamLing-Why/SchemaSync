根目录作为编排层将 schemasync-backend（Spring Boot REST API）与 schemasync-frontend（Vue3 SPA）组合为单一发布单元：
- 运行时契约集中在 backend controller 包暴露的 REST 接口上，前端通过 src/api/request.js 发起 HTTP 调用，领域模型以 JSON 在 model/dict 与 model/diff 中定义并由双方复用语义。
- adapter 层的 DatabaseAdapter 接口统一屏蔽 MySQL/Oracle/GaussDB/GoldenDB/OceanBase/TDSQL 等方言差异，被 service 层复用；generator 层按目标方言选择具体 DDLGenerator 实现。
- 开发期 frontend 通过 vite.config.js 的 proxy 将 /api 转发至后端；生产构建后由后端静态资源映射或 Nginx 反向代理部署。
- 顶层 deploy/ 提供 application.yml、start.sh/bat 与打包产物 schemasync_1.0.3.jar，构成统一的发布单元。
