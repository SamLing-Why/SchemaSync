模块采用标准 Spring Boot 分层架构，入口为 `SchemaSyncApplication`，通过 `application.yml` + `application-{dev,prod}.yml` 按 profile 切换配置。

- adapter 层：以 `DatabaseAdapter` 接口抽象各数据库方言，`DatabaseAdapterFactory` 在 `@PostConstruct` 中通过注入的 `List<DatabaseAdapter>` 自动注册到 `ConcurrentHashMap`，新增方言只需实现接口并在工厂扫描范围内；具体实现包括 `MySQLAdapter` / `OracleAdapter` / `GaussDBAdapter` / `GoldenDBAdapter` / `OceanBaseAdapter` / `TDSQLAdapter`，`ExportOptions` 作为导出参数载体。
- model 层：`model/dict` 定义 `SchemaDictionary`（含 `ExportMetadata` + `List<TableDefinition>`）、`TableDefinition`、`ColumnDefinition`、`IndexDefinition`、`ForeignKeyDefinition` 等元数据模型；`model/diff` 定义 `SchemaDiff`/`SchemaChange`/`Severity`/`ChangeType` 差异结果模型；`model/config` 暴露 `DataSourceConfig`（支持加密密码、自定义 JDBC URL、HikariCP JSON 池配置）。
- differ 层：`SchemaDiffer` 接口 + `DefaultSchemaDiffer` 实现，对两个 `SchemaDictionary` 按表→字段→索引→外键逐层计算变更集，合并同字段多属性变化为单条 `SchemaChange`，并统计 `DiffSummary`。
- generator 层：`DDLGenerator` 接口 + `MySQLDDLGenerator` 实现，将差异转为可执行 DDL。
- formatter 层：`ExcelFormatter` / `JsonFormatter` 将导出结果序列化为不同格式。
- service 层：`SchemaExportService`、`SchemaDiffService`、`DdlGeneratorService`、`SetTableService`、`ConfigService`、`SchemaFlattener`、`SchemaDictionaryParser` 编排业务逻辑。
- controller 层：`DdlController`、`DiffController`、`ExportController`、`ConfigController`、`SetTableController` 暴露 REST API，`SpaForwardController` 转发前端静态资源。
- util 层：`ConnectionPoolManager`（HikariCP 连接池封装）+ `CryptoUtil`（AES 加解密工具）。
- config 层：`WebConfig`（CORS/拦截器）、`SwaggerConfig`（OpenAPI 文档）。

依赖方向：controller → service → (adapter | differ | generator | formatter) → model/util，adapter 仅依赖 JDBC 与 model，不反向依赖上层。
